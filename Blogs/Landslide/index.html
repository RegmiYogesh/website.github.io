Introduction

Nepal's lifeline highway,
made safer by satellite

The steep, monsoon-prone highway between Narayanghat and Kathmandu is a critical transport link — the only route connecting western Nepal and Terai to the capital. Yet it suffers frequent, devastating landslide disruptions every monsoon season.

In collaboration with the University of the Sunshine Coast (Australia), Nepal's NDRRMA, and a Nepalese geospatial partner, we propose an automated system that fuses Sentinel-1 InSAR ground displacement data with DHM rainfall forecasts to trigger timely, actionable landslide alerts.

Data Sources

Multi-source data fusion
for robust monitoring

The system ingests complementary datasets that together provide both the geophysical signal of ground movement and the meteorological trigger of rainfall.

🛰
Sentinel-1 SAR
Free, all-weather C-band synthetic aperture radar data from ESA delivering global coverage every 6 days with 24-hour data latency — ideal for deformation monitoring across all weather conditions.
C-Band · 6-Day Repeat · Free Access
🌧
DHM Rainfall Forecasts
Daily rainfall data and 24–48h forecast accumulations from Nepal's Department of Hydrology and Meteorology. DHM already issues landslide warnings when upstream rainfall exceeds critical thresholds.
24–48h Forecasts · API Access
🗺
GIS Base Layers
Detailed road-network shapefile for the Narayanghat–Kathmandu highway corridor, InSAR-derived displacement rasters, digital elevation model, and a local landslide inventory for hotspot identification.
DEM · Road Network · Inventory

InSAR Processing

PS-InSAR & SBAS:
two complementary lenses

PS-InSAR
Persistent Scatterer

Uses a single "master" image to identify stable radar reflectors — excelling in areas like road surfaces and rock outcrops with little temporal decorrelation. Produces highly accurate point-based displacement time series.

SBAS
Small-Baseline Subset

Uses many short-baseline interferograms to maximize spatial coverage, especially effective in natural terrain with vegetation cover. Complements PS-InSAR for a complete deformation picture.

Both methods tested in SARscape's InSAR toolbox with parallel/GPU execution targeting ~24h processing latency.

Processing Pipeline

01
Data Ingestion
Sentinel-1 scenes downloaded daily from ESA; orbit correction and radiometric calibration applied automatically.
02
Coregistration
All scenes coregistered to a common reference. Connection graph built for SBAS network.
03
Interferogram Formation
Differential interferograms computed; unwrapping performed to extract phase displacement signals.
04
Inversion & Time Series
PS and SBAS inversion chains run in sliding-window mode; displacement vs. time extracted at key highway points.
05
Geocoding to Highway
All outputs geocoded to WGS-84; LOS motion converted to vertical/slope-normal using local DEM for road segment mapping.

Alert Criteria

Risk fusion & multi-tier
warning thresholds

Empirical thresholds calibrated from historical monsoon data. Inspired by Umbria (Italy) studies where 48h rainfall and soil saturation closely matched InSAR-detected accelerations.

Level 0
All Clear
  • InSAR velocity within baseline bounds
  • No significant rainfall forecast
  • Highway monitoring continues passively
Level 1 — Watch
Elevated Risk
  • Either heavy rainfall OR moderate ground creep detected
  • DHM forecast exceeds preliminary threshold
  • Heightened monitoring frequency
  • Authorities placed on standby
Level 2 — Warning
Imminent Hazard
  • Both InSAR motion AND rainfall thresholds exceeded simultaneously
  • 48h rainfall forecast above DHM critical value
  • Recent InSAR velocity anomaly confirmed
  • Evacuation warnings issued to authorities

Implementation Plan

Two-year phased
rollout

Months 1–3
Planning & Setup
Stakeholder alignment with NDRRMA and District DRR committees. Data agreements with ESA and DHM. Server infrastructure procurement. SARscape license setup.
Months 4–6
Pipeline Development
InSAR processing chains (PS and SBAS) built and tested on historical Sentinel-1 archive over the corridor. Baseline displacement established.
Months 7–9
Dashboard Prototype
Web GIS interface developed. Highway segments mapped with InSAR velocity overlays. Rainfall integration and alert threshold testing begins.
Months 10–12
Pilot Testing
Live system tested through a full monsoon season. Back-testing against 2023 landslide events. Thresholds iteratively refined.
Months 13–18
System Refinement
Alert criteria calibrated from pilot results. GPU pipeline optimized. Field validation in Nepal. Stakeholder feedback incorporated.
Months 19–24
Training & Handover
Capacity building workshops for NDRRMA and local DRR officers. Full operational handover. Documentation and knowledge transfer completed.
